Variadic functions

Altough the terms parameter and argument are usually used interchangeably, they actually differ.

  • A parameter is what is beeing declared by the functions signature.
  • In contrast, an argument is the concret value which is passed to a function.

In terms of parameters and arguments, JavaScript behaves rather unique. In JS one can pass less or more arguments to a function than parameters are defined. By passing less arguments, the missing parameters are undefined. By passing more more arguments, the additional arguments can be accessed using the arguments object. This comes in handy when the number of arguments are unknown.

Here's consider the simplest example in which a function does not have any parameters at all but one argument is given.

function noParams() {
    console.log(arguments);
};

console.log(noParams(1,2,3));

This is totally valid and prints the argument object to the console - check it out. This might be used if the number of arguments need to be flexible. Compared to another interpreted programming language like Python, this would end up in a typerror: TypeError: noParams() takes 0 positional arguments but 1 was given. However, ES2015 another way to provide such functionalitiy in a more clear way: the rest parameter syntax.

function restParamFun(...Args) {
    console.log(Args);
};

console.log(restParamFun(1,2,3));

By having a look in the console, you'll notice that instead of an object now an array is printed which I think is more suitable. Tha's all it is to say.
